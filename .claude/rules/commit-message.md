# Commit Message Rules

- Single line only — no multi-line messages
- English only
- Prefix required: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`, `perf:`, `ci:`
- Be specific about what changed — not why or how it was found
  - Good: `fix: fix renderer crash on empty node graph`
  - Bad: `fix: codex review`, `fix: address PR feedback`
- Do NOT include task IDs (TASK-001), issue numbers, or ticket references in the message
- Lowercase after prefix
