# タスク仕様

## 目的

イミュータブルデータ構造と構造共有に基づくアンドゥ/リドゥシステムを実装する。操作ジャーナルによるクラッシュリカバリとジャーナルコンパクションを含む。

## 要件

- [ ] `im` crateによるイミュータブルデータ構造の導入
- [ ] GraphVersion構造体（im::HashMapベースのノード・エッジ管理）
- [ ] UndoStack実装（GraphVersionの履歴スタック、undo/redo操作）
- [ ] Operation Journal（append-onlyの操作ログ記録）
- [ ] ジャーナルリプレイによるクラッシュリカバリ機能
- [ ] 正常終了時のジャーナルコンパクション（スナップショット保存 + ジャーナル切り詰め）

## 受け入れ基準

- undo/redoが構造共有により低メモリで動作（1000回操作後もメモリ使用量線形増加しない）
- ジャーナルファイルから任意時点のグラフ状態を復元可能
- クラッシュシミュレーション後にジャーナルリプレイで最終状態復元
- コンパクション後のジャーナルサイズが元の1/10以下
- undo/redo操作のレイテンシが1ms以内
