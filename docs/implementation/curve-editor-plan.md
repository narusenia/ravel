# カーブエディタ実装計画（REQ-UI-012）

## 問題

キーフレームのイージングは `Keyframe.tangent_in/out` としてデータモデルに
存在するが、編集 UI がない（Timeline の ◆ は位置と値のみ）。ベジェハンドルを
直接編集するグラフエディタが必要。要件と設計判断は
`docs/requirements/REQ-UI.md` の REQ-UI-012 に確定済み（2026-07-21 設計
セッション。Timeline 内切替ビュー採用の根拠、非破壊クランプ方針、
共有ウィジェット抽象の狙い込み）。

前提となる既存基盤: `Keyframe`（tangent_in/out + Bezier/Linear/Step、
モデル変更不要）、`interpolation::bezier`（コントロール点 x の区間クランプで
時間単調性を評価側が保証）、Timeline のキーフレーム編集（#75）・複数選択
（#105）・ppf ズーム（#108）・playhead/スクラブ、Document スナップショット
undo、lucide アイコン vendoring 方式。

## 目標アーキテクチャ

- **カーブ編集ウィジェット**（`crates/ravel-app/src/widgets/curve_editor.rs`
  想定）: 軸の意味を知らない正規化座標系のカーブ描画 + 点/ハンドル
  ヒット + ドラッグ編集部品。座標変換（データ空間 ↔ ウィジェット空間）、
  ヒット判定、ドラッグ状態遷移は GPUI 非依存の純関数群に分離して
  headless テストする。描画は Node Editor painting と同様の直接 canvas。
  gpui-component には依存しない。
- **Timeline 統合**: タイムライン右ペインを「バービュー / グラフビュー」の
  切替にする。時間→x 変換・playhead・スクラブ・キーフレーム選択集合は
  Timeline の既存状態をそのまま使う（別同期なし）。左ペインの選択
  プロパティ行が表示カーブを決める（複数選択 = 重ね描き、チャンネル
  ごとの色分けパレット）。値→y はビュー内の値レンジ（オートフィット +
  縦ズーム/パン）から変換する。
- **編集の書き込み**: 既存のキーフレーム編集経路（Document 更新、
  1 ドラッグ = 1 undo のデバウンス/commit 規約）に乗せる。ハンドルの
  ドラッグ確定値は現在の隣接区間へクランプして保存し、既存ハンドルの
  保存値は他点移動で区間が縮んでも書き換えない（評価側クランプに委ねる）。
- **描画 = 評価**: 曲線は `KeyframeCurve` の評価をフレーム刻みで
  サンプリングして折れ線描画する（独自ベジェ描画はしない）。

## 実装単位

1. **カーブ編集ウィジェット core**: 正規化座標系・カーブサンプリング
   描画・点/ハンドルヒット・ドラッグ編集コールバック。数学部の
   headless テスト。Timeline にもツールにも依存しない独立部品
   （並行委譲可能）。
2. **Timeline 統合**: グラフ切替トグル（ヘッダ、lucide アイコン追加）、
   右ペイン切替、プロパティ行セレクタと色分け、時間軸/playhead/選択の
   共有、値軸オートフィット + 縦ズーム/パン、ロケール。
3. **編集操作一式**: 点移動（Shift 軸拘束）、ハンドル対称/Alt 分離 +
   クランプ保存、補間切替右クリックメニュー、ダブルクリック追加、
   Delete 削除、ボックス選択。受入条件の編集項目を満たす。

v1.5（本計画のスコープ外、需要次第）: イージングプリセット、
ボックススケール、正規化ビュー。値-値カーブ（remap ノード +
`ParameterValue::Curve`、journal bump 案件）は別計画とし、単位 1 の
ウィジェットを再利用する。

## 着手順

ツールシステム（`docs/implementation/tool-system-plan.md`）を先に実装する。
単位 1 は独立部品なので、ツールシステム実装中の並行委譲候補。

## 完了条件（フェーズ別）

- 単位 1: ウィジェット単体でカーブ描画・ヒット・ドラッグ編集が
  headless テストで検証できる。描画サンプリングが `KeyframeCurve::evaluate`
  と一致する。
- 単位 2: 切替・重ね描き・時間軸/playhead/選択共有・フィットが動作し、
  バービューの既存挙動が不変。
- 単位 3: REQ-UI-012 受入条件をすべて満たす。編集が undo / redo /
  保存・再読込で破綻しない。
