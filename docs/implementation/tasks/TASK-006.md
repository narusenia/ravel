# TASK-006: GPUIアプリケーションシェル
- **マイルストーン**: MS1 Foundation
- **関連要件**: REQ-UI-001, REQ-UI-005, REQ-UI-007, REQ-UI-009, REQ-INFRA-001
- **規模**: L
- **依存タスク**: TASK-005

## 概要

GPUIフレームワークを用いてRavelのアプリケーションシェルを構築する。メインウィンドウ、メニューバー、ワークスペースパネルシステム、キーバインド機構、マルチウィンドウ対応を実装し、以降のUI機能（タイムライン、ノードエディタ、ビューア等）を配置するための枠組みを提供する。

## 実装ステップ

1. **GPUIアプリケーションブートストラップ**
   - `gpui::App`の初期化
   - メインウィンドウの作成（タイトル、初期サイズ、最小サイズ設定）
   - アプリケーションライフサイクル管理（起動、終了、フォーカス制御）
   - macOS: メニューバーのネイティブ統合
   - アプリアイコン設定

2. **メニューバーの実装**
   - ファイルメニュー（新規、開く、保存、名前を付けて保存、最近のプロジェクト、終了）
   - 編集メニュー（元に戻す、やり直し、カット、コピー、ペースト）
   - 表示メニュー（パネルトグル: タイムライン/ノードグラフ/ビューア/プロパティ/カーブエディタ/スコープ）
   - ワークスペースメニュー（プリセット切り替え: Edit/Node/Color/Motion）
   - ヘルプメニュー（バージョン情報）
   - メニューアクション → コマンドシステム連携

3. **ワークスペースパネルシステムの実装**
   - `gpui_component`のdock/sheet機能を基盤にパネル配置
   - パネル: タイムライン、ノードグラフ、ビューア、プロパティ、カーブエディタ（全て空のシェル状態）
   - パネルのリサイズ（ドラッグハンドル）
   - パネルの表示/非表示トグル
   - パネルレイアウトの状態管理

4. **ワークスペースプリセットシステム**
   - プリセット定義（JSON/TOML形式）
   - デフォルトプリセット4種:
     - **Edit**: タイムライン（下）+ ビューア（右上）+ メディアビン（左）+ プロパティ（右）
     - **Node**: ノードグラフ（中央）+ ビューア（右上）+ プロパティ（右下）+ カーブエディタ（下）
     - **Color**: ビューア（上）+ スコープ群（右）+ ノードグラフ（下）
     - **Motion**: ノードグラフ（中央）+ ビューア（上）+ カーブエディタ（下）+ テキストエディタ（右）
   - プリセット切り替え時のスムーズなレイアウト遷移
   - カスタムプリセットの保存/復元

5. **表示メニューからのパネルトグル**
   - 各パネルの表示/非表示をメニューから制御
   - メニュー項目にチェックマーク表示（表示中パネル）
   - キーボードショートカット対応

6. **キーバインドシステムの実装**
   - キーバインド定義ファイル（JSON/TOML）のパーサー
   - デフォルトキーバインド定義
   - キーイベント → コマンドID のマッピング
   - コマンドレジストリ（コマンドID → アクション関数）
   - 修飾キー対応（Ctrl/Cmd, Shift, Alt, Super）
   - キーコンフリクト検出

7. **マルチウィンドウ対応（パネルデタッチ）**
   - パネルをメインウィンドウからデタッチして新規ウィンドウに表示
   - デタッチウィンドウのクローズ時にパネルをメインウィンドウに復帰
   - ウィンドウ間でのデータ同期（同一`Model`を共有）
   - マルチモニタ配置の記憶・復元

8. **Propertiesインスペクタパネル（空シェル）**
   - パネル枠のみ実装（選択ノードのプロパティ表示は後続タスク）
   - 「ノードが選択されていません」プレースホルダー表示
   - パネルヘッダ（タイトル、折りたたみボタン）

9. **macOS + Windows動作確認**
   - macOS: ネイティブメニューバー、Retina対応、Cmd修飾キー
   - Windows: ウィンドウ内メニュー、DPI対応、Ctrl修飾キー
   - 両環境でパネルレイアウトが正常に動作すること
