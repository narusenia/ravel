# TASK-050: コードホットリロード
- **マイルストーン**: MS6 Pro Features
- **関連要件**: REQ-CODE-003
- **規模**: M
- **依存タスク**: TASK-048

## 概要
`.ravprj` 内スクリプトファイルの監視 → 再ロード → 該当ノード以下の dirty 化。
パラメータ宣言差分のマイグレーション（キーフレーム/接続の保持）、エラー時の
前回結果表示を実装する。外部エディタでの編集を一次体験とする。

## 実装ステップ
1. プロジェクト内スクリプトの保存形式（.ravprj エントリ + 展開時パス）
2. ファイルウォッチ（Tokio、REQ-CORE-005 の I/O 分離）→ dirty 伝播
3. パラメータ宣言差分の突き合わせ（名前+型一致で保持、削除は接続解除）
4. エラー時フォールバック（前回正常結果 + エラーバッジ）
5. 統合テスト（編集→反映レイテンシ、宣言変更マイグレーション）

## 対象コンポーネント
- `crates/ravel-app/src/project/`（保存形式）
- `crates/ravel-core/src/runtime/`（ウォッチャ）
- `crates/ravel-nodes/src/code/`

## 完了条件
- [ ] 外部エディタでの保存が自動反映される（UI 非ブロック）
- [ ] パラメータ宣言変更後もキーフレーム・接続が可能な限り保持される
- [ ] エラー時に前回結果 + エラー表示でクラッシュしない
