# REQ-CODE — コードベースジェネレーター

motion-canvas / manim 的な「コードでコンテンツを生成する」ワークフロー。
エディタを拡張する REQ-PLUGIN（Extension）とは区別し、**プロジェクト内の
コンテンツ**としてコードを扱う。コードは Layer ソース/ノードとして DAG に
参加し、既存のエフェクト・属性・フィールド・統一チャネルと相互接続できる。

## REQ-CODE-001: コード Layer / コードノード

- **優先度**: Should
- **ステータス**: Draft
- **説明**: Lua スクリプト（REQ-PLUGIN-003 と同一の mlua サンドボックス）を
  Layer ソースおよびノードとして実行する。コアの契約は**時間純関数**:
  `render(ctx) -> Geometry | FrameBuffer`（ctx: frame/time/解像度/入力値）。
  評価は Hybrid Pull（REQ-CORE-002）にそのまま乗り、キャッシュ・スクラブと
  整合する。スクリプトは `params { ... }` でパラメータを型付き宣言し、
  Properties パネル表示・キーフレーム・式・フィールド接続（統一チャネル
  REQ-CORE-007）の対象になる。ジオメトリ API（REQ-CORE-010 の属性読み書き、
  REQ-CORE-012 のフィールドサンプル）を Lua から利用できる。
- **受入条件**:
  - [ ] Lua コードを Layer ソースとして配置し、時間関数として評価される
  - [ ] Geometry / FrameBuffer の両方を出力できる
  - [ ] 宣言したパラメータが Properties に表示されキーフレーム可能
  - [ ] コードから属性 API・フィールドサンプルを利用できる
  - [ ] 実行は REQ-PLUGIN-003 と同一のサンドボックス制約に従う
  - [ ] スクリプトエラーが該当 Layer のエラー表示になり評価全体を落とさない
- **依存**: REQ-PLUGIN-003, REQ-CORE-002, REQ-CORE-007, REQ-CORE-010, REQ-CORE-012

## REQ-CODE-002: シーケンス糖衣 API

- **優先度**: Should
- **ステータス**: Draft
- **説明**: manim / motion-canvas 的な逐次記述（tween / wait / stagger /
  ease 連結）を宣言的に並べる `timeline` ビルダを標準ライブラリとして提供
  する。ビルダは内部で時間純関数（REQ-CODE-001 の契約）へコンパイルされ、
  逐次実行やベイクを行わない（ライブ編集・スクラブ耐性を保つ）。イージング
  はアニメーションカーブ（REQ-CORE-007）と同一実装を共有する。
- **受入条件**:
  - [ ] tween / wait / 並列 / stagger をコードで宣言できる
  - [ ] 生成されたアニメーションがスクラブ・巻き戻しで破綻しない
  - [ ] イージング関数群が組み込みで使える
  - [ ] 糖衣を使わない素の時間関数記述も同一契約で動く
- **依存**: REQ-CODE-001

## REQ-CODE-003: ホットリロード

- **優先度**: Should
- **ステータス**: Draft
- **説明**: スクリプトは `.ravprj` 内のファイルとして保存し、外部エディタで
  編集する（motion-canvas の dev-server 体験）。ファイル変更を監視して
  再ロードし、該当ノード以下を dirty 化して再評価する。パラメータ宣言の
  差分（追加/削除/型変更）を既存のキーフレーム・接続と突き合わせ、可能な
  限り保持する。コンパイル/実行エラーは前回の正常結果を表示しつつエラーを
  通知する。将来、簡易組み込みエディタパネルを追加検討（初期は対象外）。
- **受入条件**:
  - [ ] ファイル保存で自動再ロード・再評価される（UI 非ブロック）
  - [ ] パラメータ宣言変更後もキーフレーム・接続が可能な限り保持される
  - [ ] エラー時に前回結果 + エラー表示となりクラッシュしない
- **依存**: REQ-CODE-001, REQ-CORE-005, REQ-PROJ-001

## REQ-CODE-004: 多言語ランタイム（WASM）

- **優先度**: Could
- **ステータス**: Draft
- **説明**: REQ-PLUGIN-002 の WASM 基盤に相乗りし、REQ-CODE-001 と同一の
  API 契約（時間純関数 + パラメータ宣言 + ジオメトリ API）を WASM ABI で
  提供する。Rust / AssemblyScript 等をコード Layer の実装言語として使える
  ようにする。API 設計は言語非依存に保つ（Lua 固有機能に依存しない）。
- **受入条件**:
  - [ ] WASM モジュールがコード Layer として動作する
  - [ ] Lua 版と同一のサンプルが WASM 版でも同一出力になる
- **依存**: REQ-CODE-001, REQ-PLUGIN-002
