# REQ-DATA — 外部データ駆動

データビジュアライゼーション/ジェネラティブアート向けの、外部データによる
アニメーション・ジオメトリ駆動（Cavalry の差別化領域に相当）。

## REQ-DATA-001: テーブルデータ入力ノード

- **優先度**: Should
- **ステータス**: Draft
- **説明**: CSV / TSV / JSON ファイルを読み込み、行×列のテーブル型としてノードグラフに流す入力ノード。列は型推論（数値/文字列/ブール）され、明示上書きも可能。ファイル変更のウォッチと再読み込みに対応。パースは評価スレッドをブロックしない（REQ-CORE-005 の I/O 分離）。
- **受入条件**:
  - [ ] CSV / TSV / JSON をテーブル型として読み込める
  - [ ] 列型の自動推論と明示指定ができる
  - [ ] ファイル更新時に自動再読み込みされ下流が dirty になる
  - [ ] 不正な入力がエラー表示になり評価がクラッシュしない
- **依存**: REQ-CORE-003, REQ-CORE-005

## REQ-DATA-002: データ→属性バインディング

- **優先度**: Should
- **ステータス**: Draft
- **説明**: テーブルの行をジオメトリ要素（ポイント/インスタンス）に対応付け、列をジオメトリ属性（REQ-CORE-010）へマッピングするノード。行数に応じたインスタンス生成（データ駆動複製）と、既存ジオメトリへの属性付与の両方向をサポート。値のリマップ（範囲変換、カーブ適用）はフィールド（REQ-CORE-012）と統一チャネル（REQ-CORE-007）に委譲する。
- **受入条件**:
  - [ ] テーブル行数に応じてインスタンスを生成できる
  - [ ] 任意の列を任意名の属性としてバインドできる
  - [ ] バインドされた属性が per-instance 変調（REQ-MOGRAPH-001）に使える
  - [ ] データ更新がアニメーションに反映される
- **依存**: REQ-DATA-001, REQ-CORE-010, REQ-CORE-012

## REQ-DATA-003: リアルタイム外部入力

- **優先度**: Could
- **ステータス**: Draft
- **説明**: OSC / WebSocket / HTTP ポーリング等のライブデータソースを統一アニメーションチャネル（REQ-CORE-007）の値ソースとして接続する。ライブ入力はレンダリング時に記録済みデータへ焼き込み（bake）可能とし、最終レンダの決定性を保つ。初期リリース必須ではない。
- **受入条件**:
  - [ ] 少なくとも1種のライブソース（OSC 想定）を値ソースとして接続できる
  - [ ] ライブ値をキーフレームに bake できる
  - [ ] 切断・未受信時にフォールバック値が使われる
- **依存**: REQ-CORE-007, REQ-CORE-005
