# REQ-GPU — GPU計算パイプライン

## REQ-GPU-001: wgpu基盤 + ネイティブフォールスルー

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 通常のシェーダ処理はwgpuで統一。ハードウェアデコード（macOS VideoToolbox、Windows NVDEC/AMF等）やベンダー固有最適化が必要な箇所だけネイティブAPIにフォールスルー。GPUIがwgpuカスタムフォーク + プラットフォーム固有バックエンド（Metal/D3D11）を使用しているため、UI描画とコンピュートでGPUコンテキストを共有可能。
- **受入条件**:
  - [ ] wgpuコンピュートシェーダでノード評価が実行される
  - [ ] GPUIとGPUコンテキストが共有される
  - [ ] macOSでVideoToolbox経由のHWデコード出力をGPUメモリ上でゼロコピーで受け取れる
  - [ ] WindowsでNVDEC/AMF出力をGPUメモリ上で受け取れる
- **依存**: REQ-UI-001

## REQ-GPU-002: Hybridシェーダ管理

- **優先度**: Should
- **ステータス**: Draft
- **説明**: ビルトインシェーダはビルド時にコンパイルしてバイナリに埋め込み（起動速度確保）。ユーザーカスタムWGSLシェーダ（REQ-GPU-003）はランタイムコンパイル。開発ビルド（`#[cfg(debug_assertions)]`）では全シェーダがランタイムコンパイル + ホットリロード対応。
- **受入条件**:
  - [ ] リリースビルドでビルトインシェーダがバイナリに埋め込まれている
  - [ ] ユーザーシェーダがランタイムでコンパイル・実行される
  - [ ] 開発ビルドで全シェーダのホットリロードが機能する
  - [ ] コンパイル済みシェーダのディスクキャッシュが機能する

## REQ-GPU-003: WGSLカスタムシェーダノード

- **優先度**: Should
- **ステータス**: Draft
- **説明**: ユーザーがWGSLを記述してカスタムエフェクトノードを作成できる。入出力テクスチャとユニフォームパラメータを宣言的に定義し、Ravelが実行環境を提供。Shadertoy的な体験。
- **受入条件**:
  - [ ] シェーダノードをノードグラフに追加できる
  - [ ] WGSLコードエディタ（シンタックスハイライト付き）が提供される
  - [ ] 入力テクスチャ、出力テクスチャ、ユニフォームパラメータを宣言できる
  - [ ] ユニフォームパラメータが統一アニメーションチャネル（REQ-CORE-007）に接続可能
  - [ ] コンパイルエラーがユーザーにわかりやすく表示される
- **依存**: REQ-GPU-001, REQ-GPU-002, REQ-CORE-007
