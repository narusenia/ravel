# REQ-INFRA — インフラストラクチャ

## REQ-INFRA-001: macOSリード + Windows設計考慮、Linux後追い

- **優先度**: Must
- **ステータス**: Draft
- **説明**: macOSを主要開発プラットフォームとし最優先で品質確保。ただしWindows対応を設計段階から考慮し同時期にリリース。Linuxは後追い。コアロジックはwgpu + FFmpegでプラットフォーム非依存。プラットフォーム固有コード（HWデコーダ、オーディオ、ファイルパス等）は抽象化レイヤーで分離。
- **受入条件**:
  - [ ] macOS (Metal) で全機能が動作する
  - [ ] Windows (D3D11) で全機能が動作する
  - [ ] プラットフォーム固有コードが抽象化レイヤーで分離されている
  - [ ] （将来）Linux (Vulkan) で動作する

## REQ-INFRA-002: 英語+日本語、i18n基盤初期導入

  - [ ] 英語ロケールが完備されている
  - [ ] 日本語ロケールが完備されている
  - [ ] 言語切り替えが設定から可能
  - [ ] ハードコードされた文字列がUIに存在しない

## REQ-INFRA-003: 自動アップデート + チャネル制

- **優先度**: Should
- **ステータス**: Draft
- **説明**: アプリ起動時にアップデートを確認。Stable/Beta/Nightlyのチャネルを設け、ユーザーが選択可能。macOSはSparkle、Windowsは自前HTTP確認。Nightlyは CIからの自動ビルド。プラグインAPI互換のためsemverバージョニングルールを定義。
- **受入条件**:
  - [ ] 起動時にアップデート確認が行われる
  - [ ] ワンクリックでアップデートが適用できる
  - [ ] Stable/Beta/Nightlyチャネルを設定から選択できる
  - [ ] CIからNightlyビルドが自動生成される

## REQ-INFRA-004: フルテスト戦略

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ユニットテスト（コアエンジン）、インテグレーションテスト（ノードグラフ→レンダリング結果のE2E）、レンダリング回帰テスト（リファレンス画像のピクセル差分）、パフォーマンスベンチマーク（`criterion`、CIで追跡）、ファズテスト（`cargo-fuzz`、プロジェクトファイルパーサ/OFXホスト）。OpenFXプラグインホストの互換テスト含む。
- **CIパイプライン**:
  ```
  Push/PR → cargo test → integration tests → render regression → benchmark → fuzz (nightly)
  ```
- **受入条件**:
  - [ ] ユニットテストがCIで実行される
  - [ ] インテグレーションテストが動作する
  - [ ] リファレンス画像との回帰テストが動作する
  - [ ] パフォーマンスベンチマークがCIで追跡される
  - [ ] ファズテストがNightlyで実行される

## REQ-INFRA-006: ローカルログ + オプトイン匿名テレメトリ

- **優先度**: Should
- **ステータス**: Draft
- **説明**: `tracing`クレートで構造化ログをファイル出力。クラッシュレポート + 匿名使用統計（起動回数、使用ノード種別、レンダー時間等）をオプトインで収集。デフォルトOFF。初回起動時に明示的オプトイン。Sentry等でクラッシュ追跡。プライバシーポリシー明示。Beta/Nightlyチャネルのみパフォーマンスプロファイル収集をオプトイン提供。

  - [ ] クラッシュレポートが（オプトイン時）送信される
  - [ ] プライバシーポリシーがアプリ内から確認できる

## REQ-INFRA-007: Luaサンドボックス + OFXプロセス分離

- **優先度**: Must
- **ステータス**: Draft
- **説明**: Luaスクリプトはファイルシステム/ネットワークアクセスを制限したサンドボックス内で実行（`mlua`の`io`/`os`ライブラリ除外）。OFXプラグインはREQ-PROJ-002の子プロセスで隔離。Rustネイティブプラグインは信頼モデル。将来WASM導入時はWASMサンドボックスが自然にCapability-basedになる。
- **受入条件**:
  - [ ] Luaスクリプトからファイルシステムにアクセスできない
  - [ ] Luaスクリプトからネットワークにアクセスできない
  - [ ] OFXプラグインが別プロセスで実行される
  - [ ] プラグインクラッシュ時にメインプロセスが生存する

## REQ-INFRA-008: オープンコアライセンス (GPL依存回避)

- **優先度**: Must
- **ステータス**: Draft
- **説明**: コアエンジン + 基本ノード + プラグインAPIをOSS（Apache 2.0 / MIT）で公開。プレミアム機能/テンプレート/サポートを商用レイヤーで提供。GPL依存ライブラリは使用しないか、ダイナミックリンクで分離を徹底。FFmpegはLGPLダイナミックリンク。aubio（GPL）は使用せずMITライセンスの代替を採用。
- **ライセンス互換性**:
  | 依存 | ライセンス | 対処 |
  |------|-----------|------|
  | FFmpeg | LGPL 2.1 | ダイナミックリンク |
  | OCIO | BSD 3-Clause | 制約なし |
  | GPUI | Apache 2.0 / MIT | 制約なし |
  | wgpu | Apache 2.0 / MIT | 制約なし |
  | mlua | MIT | 制約なし |
  | aubio | GPL v3 | **不使用** — MIT代替採用 |
- **受入条件**:
  - [ ] コアエンジンがOSSライセンスで公開可能
  - [ ] GPL依存がないか、ダイナミックリンクで分離されている
  - [ ] FFmpegがダイナミックリンクされている
  - [ ] ライセンス表示がアプリ内に含まれる
