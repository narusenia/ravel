# REQ-LAYER — レイヤーネットワークモデル

タイムラインの **1 レイヤー = 1 ノードネットワーク**（Houdini 的）とする
モデル再定義。REQ-CORE-001 (v3) の詳細要件を規定する。
実装計画は `docs/implementation/layer-network-model-plan.md` を参照。

横断方針: **v2 送りの機能はデータモデルに予約フィールド/予約構造を
設け、永続化互換を確保する**（time_remap、track_matte、Expression、
HDA 的ネットワーク共有等）。

## REQ-LAYER-001: レイヤー = 殻 + ノードネットワーク

- **優先度**: Must
- **ステータス**: Draft
- **説明**: タイムラインの各レイヤーは「殻（シェル）」と「1 つのノード
  ネットワーク」で構成される。殻は汎用プロパティ — 時間配置
  （start_frame / in_frame / out_frame）、ビルトイン Transform
  （anchor_point / position / scale / rotation）、opacity、blend_mode、
  親子付け、solo / mute / locked、adjustment フラグ — を保持し、
  時間変換・空間変換・合成の責務を持つ。ネットワークはレイヤーの見た目を
  生成する純粋なノードグラフ（Houdini の SOP ネットワーク相当）で、
  殻の属性をネットワーク内に降ろさない（保守性・操作性の観点から
  汎用プロパティは殻に残す）。従来の LayerSource enum（Media / Solid /
  Shape / Text / PreComp / Generator / Null による構造分岐）は廃止し、
  レイヤー「種類」は作成時テンプレート（REQ-LAYER-008）に降格する。
  データモデル上、全レイヤーは同一構造とする。
- **受入条件**:
  - [ ] Layer が殻の汎用プロパティ + 所有するノードネットワーク（Graph）で定義されている
  - [ ] LayerSource enum とそれに伴う構造分岐（`comp.source.*` 等）が削除されている
  - [ ] transform / opacity / blend_mode / 時間配置 / 親子付けが殻の first-class フィールドとして機能する
  - [ ] v2 機能用の予約フィールド（time_remap、track_matte 等）がデータモデルに存在する
- **依存**: REQ-CORE-001, REQ-CORE-004

## REQ-LAYER-002: ネットワークインターフェース（In / Out ノード）

- **優先度**: Must
- **ステータス**: Draft
- **説明**: レイヤーネットワークは In ノードと Out ノードを 1 つずつ持つ。
  **In** は殻からネットワークへの注入点で、固定ポートとして
  `base_geometry`（GEOMETRY。レイヤー幅×高さの quad、Detail 属性に解像度。
  使われない場合は pull 型評価のためコストゼロ）と `t`（SCALAR。
  レイヤーローカル時間・秒、REQ-LAYER-006）と `f`（SCALAR。
  レイヤーローカルフレーム番号）を常備出力し、さらに
  ユーザーが型付きカスタムパラメータ（Float / Int / Bool / Vec2 / Vec3 /
  Color）を追加できる。カスタムパラメータは In の出力ポートとして現れ、
  ネットワーク内の任意の入力に配線可能で、殻の Properties パネルに
  自動露出し、キーフレーム可能（REQ-LAYER-004）。**Out** は
  `frame`（FRAME_BUFFER。殻の合成チェーンが消費する唯一のポートで、
  描画レイヤーでは必須）に加え、カスタム出力ポート（GEOMETRY / FIELD /
  SCALAR / COLOR 等、任意型）を追加でき、それらは Layer Ref ノード
  （REQ-LAYER-005）から参照される。
- **受入条件**:
  - [ ] In が `base_geometry` と `t` と `f` を常備出力する
        （`f` はレイヤールートの In のみ。サブネット内 In の
        ポートはサブネットのピン境界なので自動追加しない）
  - [ ] In にカスタムパラメータを追加・削除・名前変更・並び替えでき、出力ポートとして配線できる
  - [ ] カスタムパラメータが Properties パネルに表示・編集できる
  - [ ] Out の `frame` ポートが殻の合成チェーンに接続される
  - [ ] Out にカスタム出力ポートを追加できる
- **依存**: REQ-LAYER-001, REQ-CORE-010

## REQ-LAYER-003: サブネットワーク

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ネットワーク内に入れ子の Graph を持つサブネットワークノードを
  作れる。外観は通常ノード（入力ピンと出力ピンを持つ箱）で、内部に In /
  Out を 1 つずつ持つ。内部 In はサブネットの入力ポート（型付き、
  未接続時のデフォルト値持ち）を、内部 Out は出力ポート（型制約なしで
  複数可 — GeometryNode 的な多出力）を定義し、それらがそのまま外観の
  ピンになる。未接続の入力ピンは自動的にパラメータとして外観に露出する
  （Houdini の promote 相当）。入れ子の深さに制限なし。評価は親が
  サブネット出力を pull すると入力値が内部 In に束縛され内部 Out が
  評価される再帰（REQ-LAYER-007 と同じ機構）。v1 ではローカルな
  グループ化のみで、HDA 的な定義共有・インスタンス同期更新は v2 とし、
  移行パスを REQ-LAYER-009 で確保する。
- **受入条件**:
  - [ ] ノード群をサブネットワークにまとめられる
  - [ ] In / Out でカスタム入出力ポートを定義でき、外観のピンになる
  - [ ] 未接続の入力ピンがパラメータとして露出する
  - [ ] サブネットに潜って編集できる（REQ-LAYER-011）
  - [ ] コピー＆ペーストで複製できる
- **依存**: REQ-LAYER-002, REQ-LAYER-007

## REQ-LAYER-004: ネットワーク内パラメータのアニメーション

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ネットワーク内の任意のノードパラメータ、In のカスタム
  パラメータ、サブネットの露出パラメータをキーフレーム可能にする。
  パラメータの値ソースに統一アニメーションチャネル（REQ-CORE-007）の
  ChannelSource を採用する。Float はカーブ 1 本、Vec2 / Vec3 / Color は
  コンポーネント毎のチャンネル配列（殻の Transform と同型）。Int / Bool
  は v1 では定数のみ（step キーは v2）。**NodeOutput バインド**
  （パラメータをノード出力で駆動）は v1 で実装し、同一ネットワーク内の
  ノード出力に限定する。**Expression（Lua）は v2**。プロセッサは構築時に
  パラメータをキャプチャせず、評価時にそのフレームで解決済みの値を
  受け取るインターフェースに変更する（アニメーション中の毎フレーム
  再構築を防ぐ）。タイムラインは殻の Transform / Opacity に加え、
  キーフレームを持つネットワーク内パラメータをプロパティツリーとして
  列挙する。
- **受入条件**:
  - [ ] 任意のノードパラメータにキーフレームを打てる
  - [ ] In のカスタムパラメータにキーフレームを打てる
  - [ ] Vec / Color パラメータがコンポーネント別チャンネルでアニメーションする
  - [ ] パラメータを同一ネットワーク内のノード出力にバインドできる
  - [ ] アニメーション中のパラメータがプロセッサ再構築なしにフレーム毎評価される
  - [ ] タイムラインがネットワーク内パラメータのキーフレームを表示する
- **依存**: REQ-CORE-007, REQ-LAYER-002

## REQ-LAYER-005: Layer Ref ノード（レイヤー間参照）と Null レイヤー

- **優先度**: Must
- **ステータス**: Draft
- **説明**: Layer Ref ノード（type_key: `layer.ref`）は同一コンポジション
  内の他レイヤーの Out ポート（`frame` またはカスタムポート）を参照し、
  その型の出力ピンとして提供する。パラメータは参照先レイヤー + ポート名。
  参照は **pre-transform の素のネットワーク出力**で、参照先レイヤーの殻の
  時間変換（start / in / out）を適用した上で、参照元の評価時刻における
  値を返す。循環参照はコンパイル時に検出・拒否する（PreComp 循環検出と
  同じ層）。**Null レイヤー**を「`frame` ポートを持たずマージチェーンに
  参加しないレイヤー」として再定義し、Layer Ref 経由でのみ消費される
  供給源（Geometry / Field / パラメータ供給）とする。従来の親子付け用
  Null もこの定義に包含される。post-transform 参照、他コンポジションの
  レイヤー参照、レイヤー名の文字列による動的参照は v2。
- **受入条件**:
  - [ ] Layer Ref で他レイヤーの `frame` 出力を取り込める
  - [ ] カスタムポート（Geometry 等）を参照できる
  - [ ] Null レイヤーがマージチェーンに影響せず Layer Ref から参照できる
  - [ ] 循環参照が編集時/コンパイル時に検出・拒否される
  - [ ] 参照先の殻の時間変換が適用された値が返る
- **依存**: REQ-LAYER-002, REQ-LAYER-006, REQ-CORE-008

## REQ-LAYER-006: レイヤーローカル時間評価

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ネットワークは常にレイヤーローカル時間で評価される。
  ローカルフレーム = `comp_frame - start_frame + in_frame`。時間変換は
  殻の責務で、ネットワーク境界ノードが EvalContext を書き換えて内部
  評価に渡す（スコープ付き EvalContext）。表示区間
  `[in_frame, out_frame)` の外ではネットワークを評価せず、殻は透明として
  扱う（フレーム保持はしない）。時間の内部表現は秒（f64）で統一し、
  PreComp や異 fps メディアとの対応は秒ベースでマッピングする。
  time-dependent ノード（ノイズ等、EvalContext を見るノード）は何も
  しなくてもローカル時間駆動になる。タイムリマップ / 時間伸縮 / 逆再生は
  v2（殻に `time_remap` 予約フィールド）。
- **受入条件**:
  - [ ] レイヤーをタイムライン上で移動してもネットワーク内の時間駆動アニメーションの見た目が変わらない
  - [ ] in / out トリムがネットワーク評価時刻に正しく反映される
  - [ ] 表示区間外でネットワークが評価されない
  - [ ] 異なる fps の PreComp / メディアが秒ベースで正しくマッピングされる
- **依存**: REQ-LAYER-001, REQ-LAYER-007

## REQ-LAYER-007: 評価モデル（殻コンパイル + ネットワーク再帰評価）

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 殻部分（時間変換 → Transform → Opacity → Merge、adjustment
  分岐、親子付け Transform エッジ）は synthetic ノードへのコンパイル
  方式を維持し、決定論的 ID とキャッシュ維持の利点を保持する。
  レイヤーネットワークは**平坦化せず**、ネットワーク境界ノードが内部
  Graph の Out を再帰的に pull 評価する（サブネットも同じ機構の再帰）。
  evaluator は Graph 単体ではなく Document（Composition 解決コンテキスト）
  を受け取り、境界ノード・Layer Ref が他レイヤーのネットワークを解決
  できる。ネットワーク内ノードの評価キャッシュは所有パス
  （`CompId / LayerId / [SubnetNodeId ...] / NodeId`）のハッシュをキーと
  する。Viewer は root コンポジション出力を常時評価し、ネットワーク内
  選択ノードの単独プレビューも可能（display flag 的ピン留めは v2）。
- **受入条件**:
  - [ ] 殻の synthetic チェーンが決定論的 ID で生成され、再コンパイルでキャッシュが維持される
  - [ ] ネットワーク編集が殻の再コンパイルなしに即時評価へ反映される
  - [ ] サブネットの入れ子評価が再帰的に動作する
  - [ ] Layer Ref が Document 経由で参照先を解決する
  - [ ] root comp 出力が Viewer に常時評価される
- **依存**: REQ-LAYER-001, REQ-CORE-002, REQ-CORE-006

## REQ-LAYER-008: レイヤーテンプレートと作成時ネットワーク生成

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 「Solid を追加」等の操作はレイヤー種の構造分岐ではなく、
  初期ネットワークを生成する**テンプレート**となる。標準テンプレート:
  Solid（`In → Rasterize(color ← RGBColor) → Out`）、Video
  （`In → Video(asset_id) → Out`）、Shape（`Shape 系 → Rasterize → Out`）、
  Text（`Text(GEOMETRY) → Rasterize → Out`）、PreComp
  （`In → PreComp(comp_id) → Out`）、Null（空ネットワーク、`frame` なし
  Out）。作成後のネットワークは自由に編集可能で、テンプレートは初期状態を
  定義するだけである。テンプレート定義はデータ駆動とし、将来の
  ユーザー定義テンプレート（自作ネットワークの保存・再利用、
  REQ-PLUGIN-005 と統合）に備える。**Rasterize ノードは `color` 入力ピンを
  持ち**、Geometry が `CD` / `ALPHA` 属性を持つ場合は属性が優先、
  なければ `color` ピン、未接続ならデフォルト色パラメータの意味論を取る。
  Text ノードの GEOMETRY 出力（per-glyph Primitive + 属性）は
  REQ-MOGRAPH-004 の構造的前提であり、ノード自体の実装は v2。
- **受入条件**:
  - [ ] テンプレートから Solid / Video / Shape / Null レイヤーを作成できる
  - [ ] 作成後のネットワークを自由に編集できる
  - [ ] Rasterize が color 入力を受け、CD / ALPHA 属性優先で描画する
  - [ ] テンプレート定義がデータ駆動（コード埋め込みでない）である
- **依存**: REQ-LAYER-002, REQ-MOGRAPH-001, REQ-MOGRAPH-004

## REQ-LAYER-009: ネットワーク所有権と ID 体系

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ネットワーク（Graph）はオーナーが所有する入れ子構造とする。
  Layer が自身のネットワークを所有し、サブネットノードが内部 Graph を
  所有する（再帰的）。ノード ID はドキュメント内でグローバル一意とし
  （`NodeId::next` 採番、永続化は読み込み時にこの不変条件を維持）、
  評価インスタンスの区別は所有パス
  （`CompId / LayerId / [SubnetNodeId ...] / NodeId`）で表現する。
  イミュータブル構造の構造共有により、ネットワーク編集 → 新 Graph →
  新 Layer → 新 Composition → 新 Document の伝播がそのまま Document
  単位 undo になる。従来の「Document 単一の平坦 Graph + node_id 参照」
  方式（`Shape{node_id}` 等）は廃止する。将来の HDA 的ネットワーク共有
  （複数レイヤー・複数箇所から同一ネットワーク定義を参照し、定義の同期
  更新）に向け、ネットワークを独立リソース
  （`Document.networks: Map<NetworkId, Graph>`）へ移行可能な設計とし、
  永続化フォーマットはその移行を潰さない。
- **受入条件**:
  - [ ] Layer が自身の Graph を所有し、レイヤー削除でネットワークごと破棄される
  - [ ] 所有パスで任意の深さのノードを一意に特定できる
  - [ ] ネットワーク編集が Document 単位 undo で巻き戻せる
  - [ ] 単一平坦 Graph への参照（`Shape{node_id}` 等）が残っていない
- **依存**: REQ-LAYER-001, REQ-CORE-004

## REQ-LAYER-010: 調整レイヤー

- **優先度**: Should
- **ステータス**: Draft
- **説明**: 殻に adjustment フラグを持つレイヤーは、自身より下の
  レイヤースタックの合成結果を加工する。adjustment レイヤーの
  ネットワークでは In に `source`（FRAME_BUFFER）ポートが有効化され、
  下位スタックの合成結果が流れ込む。殻のコンパイルは通常の
  `Merge(background, foreground)` の代わりに
  `background' = network(background)` を挿入する。opacity はエフェクト
  強度（元 background との mix）として機能する。これは Layer Ref では
  表現できない（マージチェーンの途中状態は特定レイヤーの出力ではない）
  ため殻レベルの機能とする。
- **受入条件**:
  - [ ] adjustment レイヤーが下位レイヤーの合成結果にエフェクトを適用できる
  - [ ] opacity でエフェクト強度を調整できる
  - [ ] adjustment レイヤーより上のレイヤーに影響しない
- **依存**: REQ-LAYER-001, REQ-LAYER-007

## REQ-LAYER-011: ノードエディタのネットワークコンテキスト UI

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ノードエディタは常に 1 つのネットワーク（所有パスで特定）を
  編集する。タイムラインでレイヤーをダブルクリック（または
  「ネットワークを開く」コマンド）するとそのレイヤーのネットワークに
  切り替わり、サブネットをダブルクリックで更に潜る。パンくずバーで
  現在パスを表示し任意の深さに戻れる。殻の synthetic ノードはノード
  エディタに表示しない（`NodeMetadata.synthetic` のフィルタリングを
  実装）。レイヤー選択でノードエディタを強制切替しない（作業中の
  コンテキストを奪わない）。v1 は単一コンテキストで、複数ネットワークの
  同時表示・タブ化は v2。
- **受入条件**:
  - [ ] レイヤーのネットワークをノードエディタで開いて編集できる
  - [ ] サブネットに潜り、パンくずで戻れる
  - [ ] synthetic ノードがノードエディタに表示されない
  - [ ] タイムラインでレイヤー選択してもノードエディタのコンテキストが維持される
- **依存**: REQ-UI-002, REQ-LAYER-009
