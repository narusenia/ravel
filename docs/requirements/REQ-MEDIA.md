# REQ-MEDIA — メディアI/O・オーディオ

## REQ-MEDIA-001: FFmpeg + ネイティブHWデコーダ

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ソフトウェアデコード/エンコードはFFmpeg（LGPLダイナミックリンク）に委任。ハードウェアデコード（macOS VideoToolbox、Windows NVDEC/AMF/QSV、Linux VAAPI）はプラットフォーム固有コードで直接制御し、GPUメモリ上でゼロコピーで処理パイプラインに渡す。
- **サポートフォーマット**:
  - コンテナ: MP4, MOV, MKV, WebM
  - 映像コーデック: H.264, H.265, AV1, ProRes, DNxHR
  - 画像シーケンス: EXR, PNG, TIFF, DPX
  - オーディオ: AAC, PCM, FLAC, Opus
- **受入条件**:
  - [x] FFmpegダイナミックリンクでソフトウェアデコードが動作する
  - [ ] macOSでVideoToolboxによるHWデコードが動作する
  - [ ] WindowsでNVDEC/AMFによるHWデコードが動作する
  - [ ] HWデコード結果がGPUメモリ上でゼロコピーで利用できる
  - [x] 上記全コーデック/コンテナの読み込みが動作する
  - [x] エンコード（H.264/H.265/AV1/ProRes）が動作する
- **依存**: REQ-GPU-001

## REQ-MEDIA-002: CPAL + DSPクレート オーディオエンジン

- **優先度**: Must
- **ステータス**: Draft
- **説明**: CPALでクロスプラットフォームオーディオ出力。daspでミキシング処理、rubatoでサンプルレート変換。マルチトラックミキシング、基本エフェクト（ゲイン、フェード、EQ）、映像との同期再生。オーディオスレッドはリアルタイム優先度で独立動作（REQ-CORE-005）。将来VST3/CLAPプラグインホスティング対応。
- **受入条件**:
  - [x] オーディオ再生が映像と同期する
  - [x] マルチトラックミキシングが動作する
  - [ ] ゲイン/フェード/EQが動作する ← ゲイン・フェードは実装済み、EQは未実装（将来タスクで対応予定）
  - [x] サンプルレート変換が動作する
  - [x] オーディオ途切れなくリアルタイム再生される
- **依存**: REQ-CORE-005

## REQ-MEDIA-003: オーディオリアクティブ (FFT+ビート検出+BPM同期)

- **優先度**: Should
- **ステータス**: Draft
- **説明**: オーディオ解析ノードを提供。振幅（RMS/ピーク）、FFTスペクトル解析（周波数帯域分離：低/中/高域）、自動BPM検出、ビートトリガー、オンセット（音の立ち上がり）検出。タイムライン上にビートマーカーを自動配置。キーフレームをBPMグリッドにスナップ可能。解析結果は統一アニメーションチャネル（REQ-CORE-007）経由で任意パラメータに接続。GPL依存（aubio）は回避し、MITライセンスの代替を使用。
- **受入条件**:
  - [ ] AudioAnalysisノードが振幅を出力する
  - [ ] FFTスペクトル解析（帯域別エネルギー）が動作する
  - [ ] 自動BPM検出が動作する
  - [ ] ビートマーカーがタイムラインに表示される
  - [ ] キーフレームがBPMグリッドにスナップできる
  - [ ] 解析結果が統一アニメーションチャネルに接続できる
- **依存**: REQ-CORE-007, REQ-MEDIA-002
