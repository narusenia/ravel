# REQ-PROJ — プロジェクト管理

## REQ-PROJ-001: .ravprj zipコンテナ プロジェクトファイル

- **優先度**: Must
- **ステータス**: Draft
- **説明**: プロジェクトファイルは`.ravprj`拡張子のzipコンテナ。配布/受け渡し時は単一ファイル、編集中はワーキングディレクトリに展開して操作。git管理したい場合は展開状態のまま扱える。zip化時に`.cache/`を除外しファイルサイズを抑制。
- **内部構造**:
  ```
  project.ravprj (zip)
  ├── manifest.json          # バージョン、メタデータ
  ├── graph/
  │   ├── main.ron           # ルートノードグラフ
  │   └── subgraphs/         # サブグラフ定義
  ├── timeline/
  │   └── sequences.ron      # タイムライン永続化表現
  ├── assets/
  │   └── refs.json          # アセット参照（パス、ハッシュ）
  ├── settings.toml          # プロジェクト固有設定
  └── .cache/                # サムネイル等（zip時除外可）
  ```
- **受入条件**:
  - [ ] `.ravprj`ファイルの保存/読み込みが動作する
  - [ ] zip圧縮/展開が正常に行われる
  - [ ] ノードグラフがRON形式で永続化される
  - [ ] アセット参照が相対パスで記録される
  - [ ] `.cache/`がzip化時に除外可能
  - [ ] git diff可能なテキスト形式でグラフが保存される

## REQ-PROJ-002: ジャーナル + プロセス分離 + 自動保存設定

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 定期自動保存に加え、全操作をジャーナル（WAL的）に追記。クラッシュ後は最後の自動保存 + ジャーナルリプレイで復元。REQ-CORE-004のイミュータブルデータ構造のバージョン履歴とジャーナルが対応。レンダリングやOFXプラグイン実行を子プロセスに分離し、クラッシュがメインプロセスに波及しない設計。自動保存間隔は設定UIで変更可能（秒数/操作数/無効化）。`.ravprj`内に`.journal/`を設け、正常終了時にコンパクション。
- **受入条件**:
  - [ ] 自動保存が設定間隔で実行される
  - [ ] 自動保存間隔が設定UIで変更可能（秒数/操作数/無効化）
  - [ ] 操作ジャーナルが追記される
  - [ ] クラッシュ後にジャーナルリプレイで復元できる
  - [ ] OFXプラグインが別プロセスで実行される
  - [ ] プラグインプロセスのクラッシュがメインプロセスに影響しない
  - [ ] 正常終了時にジャーナルがコンパクションされる
- **依存**: REQ-CORE-004

## REQ-PROJ-003: 自動マイグレーション + バックアップ + 確認

- **優先度**: Must
- **ステータス**: Draft
- **説明**: `manifest.json`にフォーマットバージョンを記録。古いバージョンのプロジェクトを開く際、マイグレーション関数を順次適用して変換。マイグレーション実行前に`.ravprj.bak`として自動バックアップ。変更内容をユーザーに表示して確認を求める。不可逆な変更がある場合は警告。マイグレーション関数はバージョン連鎖（v1→v2→v3→...）で実装し、各ステップがユニットテスト可能。
- **受入条件**:
  - [ ] フォーマットバージョンがmanifestに記録される
  - [ ] 古いバージョンのプロジェクトが自動マイグレーションで開ける
  - [ ] マイグレーション前にバックアップが自動作成される
  - [ ] マイグレーション内容がユーザーに表示される
  - [ ] 不可逆変更に対して警告が表示される
- **依存**: REQ-PROJ-001

## REQ-PROJ-004: カテゴリ別設定 + プロジェクトオーバーライド

- **優先度**: Should
- **ステータス**: Draft
- **説明**: グローバル設定をカテゴリ別ファイル（general/appearance/keybindings/performance/plugins）で管理。設定UIで変更。プロジェクトごとに`settings.toml`でオーバーライド可能（カラースペース、FPS等）。設定のインポート/エクスポートで環境移行を容易に。パスはOS別に配慮（macOS: `~/Library/Application Support/Ravel/`、Windows: `%APPDATA%\Ravel\`、Linux: `~/.config/ravel/`）。`dirs`クレートで解決。
- **グローバル設定構造**:
  ```
  {config_dir}/
  ├── general.toml
  ├── appearance.toml    # テーマ設定含む
  ├── keybindings.toml
  ├── performance.toml
  └── plugins.toml
  ```
- **受入条件**:
  - [ ] グローバル設定が各カテゴリファイルに保存される
  - [ ] 設定UIで全設定を変更できる
  - [ ] プロジェクト固有設定でグローバル設定を上書きできる
  - [ ] 設定のインポート/エクスポートが可能
  - [ ] OS別の適切なパスに設定が保存される
