# REQ-RENDER — レンダリング・エクスポート

## REQ-RENDER-001: キューベースバックグラウンドレンダリング

- **優先度**: Must
- **ステータス**: Draft
- **説明**: レンダージョブをキューに積み、バックグラウンドで順次処理。編集作業と並行可能。ジョブ管理UI（進捗、一時停止/再開、キャンセル、優先度変更）を提供。レンダリングはREQ-CORE-005の評価スレッドプール + デコードプールを使用。
- **受入条件**:
  - [ ] レンダージョブをキューに追加できる
  - [ ] バックグラウンドでレンダリングが実行される
  - [ ] レンダリング中も編集操作が可能
  - [ ] ジョブの進捗がUIに表示される
  - [ ] ジョブの一時停止/再開/キャンセルが可能
- **依存**: REQ-CORE-002, REQ-CORE-005

## REQ-RENDER-002: Write Nodeノード単位中間出力

- **優先度**: Should
- **ステータス**: Draft
- **説明**: 任意のノードの出力にWrite Nodeを接続し、中間ファイルとして書き出し可能。デバッグや素材生成に使用。Write Nodeもレンダーキュー（REQ-RENDER-001）で管理。出力フォーマットはFFmpeg経由で全コーデック対応。画像シーケンス（EXR/PNG/TIFF/DPX）出力もサポート。
- **受入条件**:
  - [ ] Write Nodeをノードグラフに追加できる
  - [ ] 任意ノードの出力をファイルに書き出せる
  - [ ] 出力フォーマット（コーデック/コンテナ）を選択できる
  - [ ] 画像シーケンスとして出力できる
  - [ ] レンダーキューで管理される
- **依存**: REQ-RENDER-001, REQ-MEDIA-001

## REQ-RENDER-003: OCIO + GPU LUTカラーマネジメント

- **優先度**: Must
- **ステータス**: Draft
- **説明**: OpenColorIO（OCIO）で色変換を統一管理。ACES/Rec.709/sRGB/Rec.2020等のカラースペース変換。`.ocio`設定ファイルをプロジェクトに読み込み可能。ビューア表示用にOCIO変換をGPU 3D LUTにベイクしwgpuシェーダで適用（リアルタイム性確保）。LUT再生成は設定変更時のみ。RustからはFFI経由。
- **受入条件**:
  - [ ] `.ocio`設定ファイルを読み込める
  - [ ] 入力/作業/表示カラースペースを設定できる
  - [ ] カラースペース変換が正確に動作する
  - [ ] ビューア表示にGPU LUTが適用される
  - [ ] ACES, Rec.709, sRGB, Rec.2020がサポートされる
- **依存**: REQ-GPU-001
