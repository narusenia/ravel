## REQ-UI-003: Composition タイムライン（AEモデル）

- **優先度**: Must
- **ステータス**: Revised (v3 — レイヤーネットワークモデル)
- **説明**: タイムラインパネルは選択中の Composition 内 Layer 群を表示する。左パネルに Layer リスト（名前、solo/mute/lock、▼プロパティ展開）、右パネルに時間軸上の Layer バー（in/out 範囲）とキーフレーム菱形を表示する。Layer のプロパティ（position/scale/rotation/opacity 等）を ▼ で展開し、キーフレームの追加・移動・削除が可能。v3: レイヤーのノードネットワークをノードエディタで開いて自由な接続編集が可能（REQ-LAYER-011）。キーフレームを持つネットワーク内パラメータもプロパティツリーに列挙する（REQ-LAYER-004）。内部表現は常にノードグラフで、タイムラインUIは Composition の「ビュー」として機能する。
- **NLE的クリップ配置は廃止**: Track/Clip ベースの NLE モデルは採用しない。メディアは Layer のネットワーク内の Video ノード（REQ-LAYER-008）として配置する。複数メディアの切り替えはプリコンプで実現する。
- **受入条件**:
  - [ ] Layer の追加・削除・並べ替え（合成順序変更）が可能
  - [ ] Layer の in/out フレームをドラッグで変更（トリム）できる
  - [ ] Layer の start_frame をドラッグで変更（スリップ）できる
  - [ ] Layer のプロパティ（Transform 等）を展開しキーフレームを打てる
  - [ ] キーフレームの菱形がタイムライン上に表示される
  - [ ] キーフレームを持つネットワーク内パラメータがプロパティツリーに列挙される
  - [ ] Layer のノードネットワークをノードエディタで開ける
  - [ ] Composition 間の遷移（Outliner からダブルクリック等）が可能
  - [ ] ノードグラフでの変更がタイムライン表示に反映される
- **依存**: REQ-UI-001, REQ-CORE-001

## REQ-UI-011: Viewer ツールシステム（選択・移動・ペン・シェイプ描画）

- **優先度**: Must
- **ステータス**: Draft (v1 — 2026-07-21 設計確定)
- **説明**: Viewer 上で直接操作するツールセットを導入する。v1 は
  選択/移動 (V)・ペン (P)・矩形描画 (R)・楕円描画 (E)・ハンド (H)・
  ズーム (Z)。ツール状態は durable Global、切替は CommandId + Action、
  ツールバーは Viewer 内上部ストリップ（lucide アイコンを既存の
  vendoring 方式で追加）。どのツール中も中ボタンドラッグ = パン、
  H はタップで切替・ホールドで一時ハンド（当初案の Space 一時ハンドは
  トランスポートの再生/停止と衝突するため H ホールドへ変更）。
- **選択モデル**: 選択の正を `CanvasSelection`（NetworkPath + ノード集合）
  の durable Global に一元化し、Node Editor はパネル内部の選択状態を
  廃止してこれを読み書きする（二重状態による同期漏れの排除）。
  Viewer クリック選択のヒット対象は v1 ではアクティブレイヤー
  （Node Editor が開いている network）内の geometry ノードに限定する
  （ヒット候補の評価コストが選択駆動評価の既存経路に閉じるため）。
  他レイヤーへのフォールバック（アクティブ内ヒットなし → 他レイヤー）は
  v1.5 で拡張する。ヒットは評価済みジオメトリの AABB 近似
  （ピクセル精度ヒットは v2）。Shift+クリックでトグル追加。
- **バウンディングボックス**: 選択ノードの評価済み bounds を Viewer に
  オーバーレイ表示する（複数選択は合成 AABB + 薄い個別枠）。表示には
  レイヤー殻 transform を順方向に適用し、見た目と常に一致させる。
  編集（移動・ペン）は殻 transform が identity のレイヤーでのみ有効とし、
  非 identity では読み取り専用表示にする（原則:「表示は常に正しく、
  編集可能条件は明確に」。見た目座標での逆変換編集は v2）。
- **移動セマンティクス**: bbox ドラッグで移動できるのは「位置を自身の
  パラメータに持つノード」のみ — `center_x`/`center_y` を持つノード
  （shape 系・scatter 系）、`PathPoints` を持つノード（全点+接線の
  一括オフセット）、および直下流に既存の `geometry.transform` がある
  ノード（その translate を書く）。ツールによる暗黙のノード自動挿入は
  行わない（ノードグラフが正という原則を守る）。ドラッグ中は bbox 枠を
  ローカル予測で即時追従させ、ジオメトリ本体は背景評価に追従させる。
  1 ドラッグ = 1 undo（apply → commit、Esc で revert — Timeline の
  バー移動と同規約）。複数選択は移動可能ノード全部へ同一デルタ。
- **シェイプ描画**: ドラッグ矩形から shape.rect / shape.ellipse ノードを
  生成する（中心→center、寸法→size、Shift=正形、Alt=中心から描画）。
  生成先はアクティブレイヤーの network で、既存 rasterize の geometry
  入力が空いていれば自動接続（埋まっていれば未配線で置く。merge の
  自動挿入はしない）。レイヤーが無ければ Shape レイヤーテンプレートを
  自動作成する。polygon/star はドラッグ 1 回で決まらないため
  Add Node 経由のまま（v1 対象外）。
- **ペン**: ベジェは予約点属性 `in_tan`/`out_tan`（Vec2）で表現し、
  `Primitive::Path` は不変。点列は `ParameterValue::PathPoints`
  （`Vec<PathPoint>`、PathPoint = p/in_tan/out_tan）として
  `shape.custom_path` のパラメータに保存する（undo・RON 永続化・
  Properties は既存経路に乗る）。描画はクリック=コーナー/ドラッグ=
  スムーズ対称/始点クリックで閉路/Esc 確定、既存点・ハンドルの移動可。
  点の挿入/削除/Alt でのハンドル分離は v1.5。新規パス描画時は
  path→rasterize を自動配線し選択を切り替える。曲線の描画は
  comp 空間固定 tolerance の**共有フラット化**（CPU zeno と GPU に
  同一折れ線を渡し、GPU/CPU 等価性テストを維持）。GPU シェーダでの
  曲線直接評価は将来展望（v2 以降）。座標系は comp ピクセル空間
  （レイヤー殻の逆変換は v2）。パスアニメーション（点列の時間変化）は
  実装せず、PathChannel の設計書のみ実装計画に含める。
- **受入条件**:
  - [ ] V/P/R/E/H/Z でツールが切り替わり、ツールバーに現在ツールが示される
  - [ ] H ホールドで一時ハンド、離すと元ツールに戻る。中ドラッグは常にパン
  - [ ] 選択ノードの bbox が Viewer に表示され、殻 transform 込みで
        見た目と一致する
  - [ ] Viewer クリックでアクティブレイヤー内のシェイプを選択でき、
        Node Editor のハイライトと Properties が追従する
  - [ ] bbox ドラッグで shape / custom_path / 直下流 transform 持ちノードが
        移動でき、1 ドラッグ = 1 undo、Esc でキャンセルできる
  - [ ] ドラッグで rect / ellipse を描画でき、rasterize へ自動配線される
  - [ ] ペンでコーナー/スムーズ点の混在パスを描き、閉路・確定でき、
        点とハンドルを後から編集できる
  - [ ] 曲線が CPU / GPU 両経路で同一に描画される
  - [ ] ツール操作が undo / redo / 保存・再読込で破綻しない
- **依存**: REQ-UI-001, REQ-UI-002, REQ-UI-004, REQ-CORE-001

## REQ-UI-012: カーブエディタ（グラフエディタ）

- **優先度**: Should
- **ステータス**: Draft (v1 — 2026-07-21 設計確定)
- **説明**: キーフレームカーブ（時間-値）をベジェハンドルで編集する
  グラフエディタ。既存の `Keyframe` モデル（`tangent_in`/`tangent_out` +
  Bezier/Linear/Step）をそのまま編集対象とし、データモデルの変更はない。
  **Timeline 内の切替ビュー**（AE 方式）として実装する — タイムライン
  右ペインをヘッダのトグル（lucide アイコン）でグラフ表示に切り替え、
  時間軸スケール・ズーム・playhead・スクラブ・キーフレーム選択
  （Timeline の ◆ と同一集合）・undo（Document スナップショット、
  1 ドラッグ = 1 undo）をすべて Timeline と共有する。独立パネル間の
  時間軸/選択同期プロトコルを作らないための判断。表示対象カーブは
  左ペインの選択プロパティ行（複数選択で重ね描き、チャンネルごとに色分け）。
- **共有ウィジェット基盤**: カーブ描画・点/ハンドルのヒット・ドラッグ編集は
  「軸の意味を知らない正規化座標のカーブ編集ウィジェット」として
  gpui-component 非依存で自前実装する（独自 UI ライブラリ方針）。
  将来の値-値カーブ（カーブ版 remap ノード / `ParameterValue::Curve`、
  別計画）が同ウィジェットを Properties 埋め込みで再利用する前提の抽象。
- **WYSIWYG 原則**: エディタが描く曲線は core の `evaluate`
  （コントロール点 x の区間クランプ込み）をフレームサンプリングした
  結果そのもの — 描画とアニメーション評価のズレを構造的に排除する。
  ハンドルのドラッグ確定値は隣接キーフレーム区間内へクランプして保存する。
  ただし他の点の移動で区間が後から縮んだ既存ハンドルの保存値は
  書き換えない（評価側クランプに任せる非破壊方針。区間を戻すと
  ハンドルが復元される）。
- **v1 編集操作**: 点の移動（Shift で時間/値の軸拘束）、ハンドル編集
  （既定は対称連動・**Alt で分離** — ペンツール REQ-UI-011 と同一の
  修飾キー規約）、補間タイプ切替（Bezier/Linear/Step、選択点一括の
  右クリックメニュー）、カーブ上ダブルクリックで点追加、Delete で削除、
  ボックス選択、値軸オートフィット（表示カーブ群の合成 min/max）+
  ホイール縦ズーム/パン（水平ズームは Timeline の ppf と共有）。
  値軸は実値表示。
- **v1.5**: イージングプリセット（Easy Ease 等）、ボックススケール
  （選択群の時間/値一括伸縮）、正規化ビュー（各カーブを自身の min/max で
  0..1 に重ねる比較表示）。
- **受入条件**:
  - [ ] Timeline のトグルでグラフビューに切り替わり、時間軸・playhead・
        スクラブが Timeline と一致する
  - [ ] 選択プロパティ行のカーブが色分けで重ね描きされる
  - [ ] 点とハンドルをドラッグ編集でき、描画カーブが評価結果と常に一致する
  - [ ] ◆（Timeline）とグラフ上の点の選択が同一集合として連動する
  - [ ] 補間切替・点追加・削除・ボックス選択が機能する
  - [ ] 編集が undo / redo / 保存・再読込で破綻しない（1 ドラッグ = 1 undo）
- **依存**: REQ-UI-001, REQ-UI-003, REQ-CORE-001
